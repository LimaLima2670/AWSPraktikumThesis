# AWS-Praktikumsversuch

## Versuchsablauf des AWS-Versuchs für MVS

Dieses Repository dokumentiert den strukturierten Aufbau und die Durchführung der praktischen Versuche im Rahmen der Bachelorarbeit im Studiengang Medientechnik.

Der Fokus liegt auf der Untersuchung cloudbasierter Video-Transcoding- und Content-Auslieferungs-Workflows unter Nutzung von:

- Amazon S3  
- AWS MediaConvert  
- AWS Lambda  
- Amazon SNS  

---

## Versuche

### 🔹 [Versuch 1 – Grundlagen eines AWS-basierten Transcoding-Workflows](Versuch1/)
Einführung in die grundlegenden AWS-Dienste und Aufbau eines einfachen Video-Transcoding-Workflows mit Amazon S3 und AWS MediaConvert.

### 🔹 [Versuch 2 – Content-Auslieferung über CDN](Versuch2/)
Untersuchung der Auslieferung transcodierter Inhalte über ein Content Delivery Network unter Nutzung von AWS-Diensten.

### 🔹 [Versuch 3 – Automatisierter Serverless-Workflow](Versuch3/)
Konzeption und Umsetzung eines automatisierten Videoverarbeitungs-Workflows mit AWS Lambda und Amazon SNS.

### 🔹 [Versuch 4 – Export und CDN-Integration](Versuch4/)
Analyse der Auslieferung und Integration der erzeugten Inhalte in eine CDN-Infrastruktur.

### 🔹 [Versuch 5 – Steuerung über CLI und SDK](Versuch5/)
Untersuchung der Steuerung von AWS-Ressourcen über die AWS CloudShell, die AWS CLI sowie das Python SDK (boto3).

---

## Zielsetzung

Ziel der Versuchsreihe ist es, ein fundiertes Verständnis für:

- Cloud-Architekturen im Medienbereich  
- Video-Transcoding-Prozesse in AWS  
- Serverless-Architekturen  
- Skalierbarkeit und Performance von Cloud-Workflows  

zu entwickeln.

---

## Kontext

Das Projekt entsteht im Rahmen der Bachelorarbeit an der Hochschule RheinMain und dient der strukturierten Dokumentation der experimentellen Umsetzung.
